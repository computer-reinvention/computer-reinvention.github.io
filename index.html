experiments

Tools for a future where software is written, navigated, and reviewed with agents. Each experiment attacks the same question from a different angle: what does programming look like when humans stop reading code line by line?

01 triecontext layer Your codebase, in prose. A reference-graph cascade keeps a Markdown twin of every source file honest — drift is a build break — and coding agents navigate the prose instead of grepping code. read the story → 02 AGMattention map The Attention Gravity Map — a macOS editor where you watch, turn by turn, where your coding agent is thinking. Attention lands as mass on symbols, glows, and decays as focus moves on. watch it think → 03 flowpilotbrowser automation Explore once, replay forever. An agent walks a website once and compiles cached reflexes — warm runs replay with zero LLM calls, and humans join the loop programmatically when the web pushes back. see the loop → 04 sorenagent swarm A self-improving swarm of autonomous agents — supervisor, workers, reviewers, testers — coordinating through message passing to build software, including their own codebase. meet the swarm → 05 multiclaudeparallel dev Parallel Claude Code development. Multiple agent instances work features simultaneously in isolated worktrees and tmux windows, coordinated by a supervisor. open the lanes → 06 chainfactoryllm pipelines Declarative multi-step LLM workflows in a YAML-like syntax. Chain prompts, fan out in parallel over iterables, reduce back — split-map-reduce for inference. tour the factory →
